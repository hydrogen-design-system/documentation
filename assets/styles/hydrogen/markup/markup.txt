              <p>First, we'll need to install <a href="https://nodejs.org/en/" title="Learn more about installing Node and NPM." target="_blank">NPM</a> if you don't already have it on your machine. From there, in a command prompt/terminal window, navigate to the root of your project and enter:</p>
              <pre>npm install @hydrogen-design-system/hydrogen.css --save-dev</pre>
              <p>After the installation completes, you can run:</p>
              <pre>npx h2-init</pre>
              <p>The above command will prompt you to enter two folder locations: your markup folder and your styles/CSS folder. Simply enter the path from the root of your project (e.g. <code>src/my/markup</code>). Once you've submitted these paths, Hydrogen will automagically create a <code>hydrogen.config.json</code> file in the root of your project. <span data-h2-font-weight="b(600)" data-h2-font-color="b(theme-1)">This file will be where you can customize Hydrogen's settings and theme.</span></p>

              {% include heading-3.html name="Syntax" id="syntax" %}
              <p>To begin, let's dissect how a Hydrogen attribute works so that we can better understand how to use them.</p>
              <pre>data-h2-PROPERTY="MEDIA(ARGUMENTS)"</pre>
              <p>Hydrogen attributes have 4 distinct pieces to them:</p>
              <ul>
                <li><code>data-h2-</code>: this is how custom <code>data-attributes</code> are defined in HTML. The <code>h2</code> bit ensures that Hydrogen's attributes are namespaced and won't interfere with other frameworks.</li>
                <li><code>PROPERTY</code>: the property portion is where we specify which CSS property we want to style. <a href="#backgroundColor" title="View a list of properties available in Hydrogen.">A list can be found starting with background color below.</a></li>
                <li><code>MEDIA()</code>: this is how we define at which media query we want our styles to apply at. You can learn more about how to use media queries and which ones are installed by default below.</li>
                <li><code>ARGUMENTS</code>: each property has its own special set of arguments, outlined in the documentation below, that will define how the element is styled.</li>
              </ul>
              {% include heading-3.html name="Media Queries" id="mediaQueries" %}
              <p>Media queries in Hydrogen are designed to allow you to granularly control each property when you need it. Rather than being forced to write unique styles for a particular element, you can directly target that element's property at the exact query you need right in your markup.</p>
              <p>Hydrogen comes with a default set of 4 media queries out of the box:</p>
<pre>
b
s (48em)
m (64em)
l (100em)
</pre>
              <p>Using a media query is simple:</p>
              <pre>data-h2-bg-color="b(black) m(theme-1) l(theme-2)"</pre>
              <p>Renders something like this (try resizing your screen!):</p>
              <div data-h2-bg-color="b(black) m(theme-1) l(theme-2)" data-h2-padding="b(all, l)" data-h2-margin="b(bottom, m)"></div>
              <p>If the built in media queries don't suit your needs, you can <a href="#configMedia" title="Learn more about setting custom media values in your configuration file.">customize them</a> in your <code>hydrogen.config.json</code> file.</p>
              {% include heading-3.html name="Color and Gradients" id="colors" %}
              <p>Hydrogen comes with a set of 5 pre-installed colors, but we know that color is almost always something you're going to want to control. In this section, we want to explain how color is used in the context of property arguments, rather than highlight the colors themselves. <a href="#configThemeColors" title="Learn more about configuring colors.">You can customize the exact palette for your application by following the instructions in the Themes section.</a></p>
<pre>
black
white
purple
blue
pink
</pre>
              <p>When colors are built in Hydrogen, a light and dark version of each colors is created, and there is the option for each one to enable automatic generation of opacity variations. Hydrogen's default colors have this opacity option disabled, but these can be extremely helpful if your application frequently uses variations in its UI.</p>
              <p><span data-h2-font-color="b(theme-1)" data-h2-font-weight="b(600)">The simplest way to reference a color in a property argument is to simply use its name</span>: <code>data-h2-bg-color="b(purple)"</code>. Sometimes however, we want to tap into the automatically created lighter, darker, or transparent versions of a color, and to so, colors have some prefix and suffix syntax that can be optionally included:</p>
              <ul>
                <li><code>[dark]COLOR</code> for light/dark versions</li>
                <li><code>COLOR[.6]</code> for referencing an opacity</li>
              </ul>
              <div data-h2-flex-grid="b(top, expanded, flush, m)">
                <div data-h2-flex-item="b(1of1) m(1of2)">

              <p>These values can even be combined to target a transparent dark variation, for example: <code>data-bg-color="b([dark]purple[.5])"</code></p>
              {% include heading-4.html name="Gradients" id="gradients" %}
              <p>Hydrogen doesn't ship with any gradients enabled by default, but you can optionally configure them in the <a href="#configThemeGradients" title="Learn more about configuring gradients.">gradients section of your configuration file</a>.</p>
              <p data-h2-font-color="b(theme-1)" data-h2-font-weight="b(600)">Gradients are supported by the following properties:</p>
<pre>
bg-color
</pre>
              <p>When they are enabled, you can reference them in place of a color value by using the following syntax: <code>TYPE-ANGLE[COLORSTOP1][COLORSTOP2]</code>.</p>
              <p>More specifically, radial gradients will not accept an angle value, while linear gradients require them. These angle values are assigned in your configuration file.</p>
              <p data-h2-font-weight="b(600)">Examples:</p>
<pre>
linear-45[purple][blue]
radial[pink][purple]
linear-270[purple][blue][pink]
</pre>
              {% include heading-3.html name="Whitespace" id="whitespace" %}
              <p>Whitespace in Hydrogen works within certain constraints. In order to help ensure the flow of whitespace is consistent in your app and you aren't worrying about arbitrary values for margins, padding, and the like, Hydrogen generates a set of whitespace values for you using the <code>whitespaceScale</code> option in your configuration file.</p>
              <p>By default, the <code>whitespaceScale</code> value is set to 1.5, and this means that, starting with <code>1rem</code>, whitespace values are calculated like this:</p>
<pre>
none (0rem)
xxs (0.4444rem)
xs (0.6666rem)
s (1rem)
m (1.5rem)
l (2.25rem)
xl (3.375rem)
xxl (5.0625rem)
</pre>
              <p>These values can be used in any property that has a <code>WHITESPACE</code> argument.</p>

              <p>After running <code>npx h2-init</code>, a <code>hydrogen.config.json</code> file will be located in the root of your project. In this file, you can specify a variety of options that will modify how Hydrogen works and the theme it applies to its stylesheet.</p>

              {% include heading-4.html name="Setting Up Project Folders" id="configFolders" %}
              <p>If you have run <code>npx h2-init</code> these configuration lines are probably already set. These options indicate to Hydrogen where it should find your markup and where it should spit out its final CSS file.</p>
<pre>"folders": {
  "markup": "path/to/markup/folder",
  "styles": "path/to/css/folder"
}</pre>
              <p>While by default the folders configuration contains only one option for markup, this configuration can also be set to an array if your project has multiple markup locations:</p>
<pre>"folders": {
  "markup": [
    "first/path/to/markup/folder",
    "second/path/to/markup/folder"
  ],
  "styles": "path/to/css/folder"
}</pre>
              {% include heading-4.html name="Choosing Media Queries" id="configMedia" %}
              <p>By default, Hydrogen comes with 4 default media queries, one of which (<code>b</code> for "base") will always be available on your project. You can configure your own media queries by modifying the list below in your configuration file to suit your project's needs. If you're happy with the default options, you can omit this from your configuration file.</p>
<pre>"media": [
  {
    "name": "s",
    "value": "48em"
  },
  {
    "name": "m",
    "value": "64em"
  },
  {
    "name": "l",
    "value": "100em"
  }
]</pre>
              {% include heading-4.html name="Media Query Warnings" id="configMediaWarnings" %}
              <p>This option is set to <code>false</code> by default. Setting this to <code>true</code> will allow warning messages to appear in your console during a production build to indicate if there are media queries being used in your markup that aren't included in the <code>media</code> configuration outlined above. These messages can be helpful to find typos or queries that simply don't exist.</p>
              <pre>"mediaWarnings": true,</pre>
              {% include heading-4.html name="Debug Settings" id="configDebug" %}
              <p>This option is set to <code>false</code> by default. Setting this to <code>true</code> will prevent Hydrogen from cleaning up its cache (found in your CSS folder). This can be helpful to inspect the various output files during Hydrogen's build to locate problems or bugs.</p>
              <pre>"debug": true,</pre>

{% include heading-3.html name="Data Attributes?" %}
<p>Why use <code>data-attributes</code> over <code>class</code>?</p>
<p>There's been a lot of discussion around classes performing better than data attributes, and we took this into consideration when designing Hydrogen. A lot of traditional class practices like BEM are effective, but can result in some really complex, bloated class attributes, which can have impacts on our ability to consume and write code. One of Hydrogen's primary goals is to make the end-user experience AND the developer experience easier, and so we decided to use data attributes for a few reasons:</p>
<ul>
  <li>they provide consistent namespacing</li>
  <li>they provide a context and a value</li>
  <li>their values are flexible, and allow for modifiers</li>
  <li>they allow for complex combinations</li>
  <li>they're easy to manipulate with JavaScript</li>
</ul>
<p>In combination with Hydrogen's ability to generate a custom CSS file that contains only what you've used, the performance trade-offs are worth the improved experience. It was of the utmost importance that Hydrogen considered maintenance and legibility so that markup could be easily understood while retaining good performance and a small footprint.</p>
{% include heading-3.html name="Customization?" %}
<p>Hydrogen offers customization of both style and syntax through a comprehensive JSON configuration file. Include as many media queries, colors, gradients, and more as your project needs to get the job done. The <a href="#configuration" title="Learn more about how to configure Hydrogen.">Configuration and Themes</a> section of the documentation offers more detail about how this file works.</p>
{% include heading-3.html name="File Size?" %}
<p>When you're ready to publish your project to the world, Hydrogen's build script will process your HTML, React, etc. and find exactly the attributes your project is using. It then compiles a CSS file that is specifically designed for your project. No bloat, no unused classes or CSS. By doing so, Hydrogen's file size drops drastically, improving both page load speeds and the performance of the data-attributes themselves.</p>

{% include heading-3.html name="Compile" %}
<p><code>npx h2-compile</code> will tell Hydrogen to compile its whole CSS file once, based on your current configuration. This will output Hydrogen in its entirety, so you'll be able to add and remove markup with little friction or waiting for a compile to finish.<span data-h2-font-weight="b(600)" data-h2-font-color="b(theme-1)">This is probably the command you'll use most</span> and is helpful if you're actively developing on your project.</p>
{% include heading-3.html name="Build" %}
<p><code>npx h2-build</code> should be used when you're ready to deploy your project, as it will compile Hydrogen by searching through your markup and create a CSS file that contains only the parts of Hydrogen that you've used, rather than the whole system, thus reducing Hydrogen's file size drastically. You can learn more about this in the "<a href="#what" title="Learn about how Hydrogen creates its production file.">What is Hydrogen?</a>" section.</p>
